# Skill: Adoption Planning

> **Status:** Placeholder — not implemented in v0.1

## Purpose

The Adoption Planning skill translates readiness outputs and strategic objectives into a structured, phased adoption plan. It defines who gets access to what, in what sequence, with what support — from initial pilot through to broad deployment and sustained use.

## CAF Phase

**Plan · Adopt · Govern**

## Invoked By

- Readiness Agent
- Adoption Agent
- Governance Agent

## Inputs

- Organizational input profile (departments, user count, Copilot license count, champions program status)
- Readiness checklist outputs
- Scenario Pack: Role-based deployment sequencing

## Expected Outputs

- Phased adoption plan (pilot → broad deployment → sustained use)
- Pilot cohort definition (who, what roles, what departments)
- License deployment schedule
- Adoption milestones and success criteria
- Escalation and support plan

## Key Activities

1. Define the pilot cohort (recommended: 10–20% of total licensed users, cross-departmental)
2. Sequence deployment phases based on organizational readiness and priority departments
3. Define adoption milestones for each phase
4. Identify support structures for each phase (help desk, champions, IT support)
5. Define escalation paths for technical and adoption issues
6. Produce the phased adoption plan document

## Adoption Phases

| Phase | Description | Typical Duration |
|---|---|---|
| Pilot | Controlled deployment to a defined cohort | 4–6 weeks |
| Broad Deployment | Expansion to all licensed users | 4–8 weeks |
| Sustained Use | Ongoing optimization, new use case activation | Continuous |

## Notes

Adoption planning is not a one-time exercise. The plan should be reviewed at each phase gate and updated based on pilot learnings and organizational feedback.
