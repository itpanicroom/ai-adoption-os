# Skill: Success Measurement

> **Status:** Placeholder — not implemented in v0.1

## Purpose

The Success Measurement skill defines the framework for measuring whether AI adoption is working. It establishes the KPIs, data collection approach, and reporting cadence that allow the organization to demonstrate value, identify issues early, and make informed decisions about the next adoption phase.

This skill produces measurement frameworks — not maturity scores. The focus is on business outcomes and adoption indicators, not ratings.

## CAF Phase

**Govern · Manage**

## Invoked By

- Governance Agent
- Measurement Agent

## Inputs

- Organizational input profile
- Adoption plan (deployed use cases and roles)
- Scenario Pack: Technology-specific success indicators
- Cookbook: Manage phase measurement guidance

## Expected Outputs

- Success metrics framework document
- KPI definitions for each deployed use case
- Baseline measurement plan (how to establish a pre-deployment baseline)
- Reporting template and cadence plan
- Executive dashboard outline (high-level metrics for leadership)

## Key Activities

1. Define the business outcomes the organization expects from this AI deployment
2. Map expected outcomes to measurable indicators
3. Establish a pre-deployment baseline for each metric (where possible)
4. Define data sources for each metric
5. Design the reporting cadence and format
6. Identify thresholds that trigger a review or corrective action

## Metric Categories

| Category | Examples |
|---|---|
| **Adoption metrics** | Active users, feature usage rate, sessions per user per week |
| **Productivity metrics** | Time saved per task, documents drafted, meetings summarized |
| **Quality metrics** | User satisfaction score, error rates, output quality ratings |
| **Business metrics** | Deal velocity (Sales), time-to-hire (HR), report cycle time (Finance) |
| **Governance metrics** | Policy compliance rate, data classification coverage |

## Notes

Measurement is most effective when baselines are established before deployment. Organizations should capture current-state metrics during the Plan phase so that post-deployment comparisons are meaningful.
